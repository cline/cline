```bash
cline --help          # Show all commands

```

## Synopsis

```bash
cline [prompt] [options]
cline <command> [options] [arguments]
```

## Global Options

| Option | Description |
|--------|-------------|
| `-P, --provider <id>` | Provider ID (default: `cline`) |
| `-k, --key <api-key>` | API key override |
| `-m, --model <id>` | Model to use (e.g., `claude-sonnet-4-6`, `gpt-4o`) |
| `-a, --act` | Run in act mode (default) |
| `-p, --plan` | Run in plan mode (planning only, no execution) |
| `-y, --yolo` | Auto-approve all actions, plain text output, exit on completion |
| `-i, --interactive` | Start interactive chat mode |
| `-s, --system <prompt>` | Override the system prompt |
| `-c, --cwd <path>` | Set working directory |
| `-n, --max-iterations <count>` | Maximum agent iterations |
| `-t, --timeout <seconds>` | Timeout in seconds |
| `-T, --taskId <id>` | Resume an existing task by ID |
| `--config <dir>` | Custom configuration directory |
| `--json` | Output as newline-delimited JSON |
| `-u, --usage` | Show token usage and estimated cost |
| `--timings` | Show timing details |
| `-v, --verbose` | Verbose output |

### Tool Control

| Option | Description |
|--------|-------------|
| `--tools` | Enable tools (default) |
| `--no-tools` | Disable all tools |
| `--autoapprove <value>` | Set auto-approval for all tools (`true` or `false`) |
| `--auto-approve-all` | Auto-approve all actions while keeping interactive mode |
| `--tool-enable <name>` | Enable a specific tool (repeatable, comma-separated) |
| `--tool-disable <name>` | Disable a specific tool (repeatable, comma-separated) |
| `--max-consecutive-mistakes <n>` | Max consecutive mistakes before halting in yolo mode |

### Reasoning

| Option | Description |
|--------|-------------|
| `--thinking` | Enable extended thinking (default: medium effort) |
| `--reasoning-effort <level>` | Set reasoning effort: `none`, `low`, `medium`, `high`, `xhigh` |

### Agent Teams and Spawning

| Option | Description |
|--------|-------------|
| `--teams` | Enable agent team tools (default) |
| `--no-teams` | Disable agent team tools |
| `--team-name <name>` | Set team name for team workflows |
| `--spawn` | Enable sub-agent spawning (default) |
| `--no-spawn` | Disable sub-agent spawning |
| `--enable-spawn` | Force enable spawning (overrides other settings) |

### Sandbox

| Option | Description |
|--------|-------------|
| `--sandbox` | Use isolated local state instead of `~/.cline` |
| `--sandbox-dir <dir>` | Custom sandbox directory (default: `/tmp/cline-sandbox`) |

### Hooks

| Option | Description |
|--------|-------------|
| `--hooks-dir <dir>` | Path to additional hooks directory |

## Commands

### cline (default)

Start a task or enter interactive mode.

```bash
cline                              # Interactive mode
cline "your prompt here"           # Start a task
cline -y "do something"            # Yolo mode
echo "prompt" | cline              # Piped input
```

### cline task (alias: t)

Run a task with a prompt. Equivalent to `cline "prompt"`.

```bash
cline task "Create a REST API endpoint"
cline t "Fix the bug in utils.js"
```

### cline auth

Configure authentication with an AI provider.

```bash
cline auth                         # Interactive wizard
cline auth -p anthropic -k sk-ant-... -m claude-sonnet-4-6
cline auth -p openai-native -k sk-... -m gpt-4o
cline auth -p openai -k key -b https://api.example.com/v1
```

The interactive wizard supports OAuth sign-in (Cline, ChatGPT subscription, OCA) and manual API key entry for any provider.

| Option | Description |
|--------|-------------|
| `[provider]` | Provider ID (positional shorthand for `-p`) |
| `-p, --provider <id>` | Provider ID |
| `-k, --apikey <key>` | API key |
| `-m, --modelid <id>` | Model ID |
| `-b, --baseurl <url>` | Base URL (OpenAI-compatible providers only) |

### cline connect

Connect to messaging platforms. See [Connectors](/cli/connectors).

```bash
cline connect                      # Interactive wizard
cline connect telegram -m <BOT-ID> -k <BOT-TOKEN>
cline connect --stop               # Stop all connectors
cline connect <adapter> --stop     # Stop specific connector
```

### cline mcp

Manage MCP servers. See [MCP Servers](/cli/mcp-servers).

```bash
cline mcp                          # Interactive wizard
```

### cline schedule

Manage scheduled agents. See [Scheduling](/cli/scheduling).

```bash
cline schedule                     # Interactive wizard
cline schedule create "Name" --cron "0 9 * * MON-FRI" --prompt "Task"
cline schedule list
cline schedule trigger <id>
cline schedule pause <id>
cline schedule resume <id>
cline schedule delete <id>
cline schedule executions <id>
```

### cline history (alias: h)

Browse and manage session history.

```bash
cline history                      # Interactive session selector
cline history --json               # List as JSON
cline history --limit 50           # Limit results
cline history delete --session-id <id>
cline history update --session-id <id> --title "New Title"
```

| Option | Description |
|--------|-------------|
| `--json` | Output as JSON |
| `--limit <n>` | Maximum sessions to show (default: 200) |
| `--page <n>` | Page number for pagination |
| `--session-id <id>` | Session ID for update/delete |

### cline checkpoint

Inspect or restore session checkpoints.

```bash
cline checkpoint status --session-id <id>
cline checkpoint list --session-id <id>
cline checkpoint restore latest --session-id <id>
cline checkpoint restore 1 --session-id <id>    # Restore by index
cline checkpoint restore latest -y              # Skip confirmation
```

| Option | Description |
|--------|-------------|
| `--session-id <id>` | Target session (required) |
| `-y, --yes` | Skip confirmation prompt |
| `--json` | Output as JSON |

### cline config

View and manage configuration.

```bash
cline config                       # Interactive config browser
cline config workflows             # List workflows
cline config rules                 # List rules
cline config skills                # List skills
cline config agents                # List configured agents
cline config plugins               # List plugins
cline config hooks                 # List hooks
cline config mcp                   # List MCP servers
cline config tools                 # List available tools
cline config --json <target>       # Output as JSON
```

### cline hub

Manage the hub daemon. The hub starts automatically when `ClineCore` needs it, so most users don't need these commands. Use `cline hub stop` to shut down a hub running in the background.

```bash
cline hub start                    # Start the hub daemon
cline hub stop                     # Stop it
cline hub status                   # Check if running
cline hub ensure                   # Start if not running, return URL
```

### cline doctor

Diagnose configuration and connectivity issues.

```bash
cline doctor
```

Checks hub health, active processes, connector status, hook workers, and stale locks.

### cline dev

Developer tools.

```bash
cline dev log                      # Open the CLI log file
```

### cline version

```bash
cline version
cline -V
```

## Environment Variables

| Variable | Description |
|----------|-------------|
| `CLINE_DATA_DIR` | Custom configuration directory (replaces `~/.cline/data/`) |
| `CLINE_HUB_ADDRESS` | Override hub address (default: `127.0.0.1:25463`) |
| `CLINE_SESSION_BACKEND_MODE` | Force backend mode (`local`, `hub`, `remote`, `auto`) |
| `CLINE_SANDBOX_DATA_DIR` | Sandbox session storage directory |
| `CLINE_SANDBOX` | Enable sandbox mode |
| `CLINE_HOOKS_DIR` | Additional hooks directory |
| `CLINE_BUILD_ENV` | Set to `development` for debug features |
| `CLINE_DEBUG_PORT_BASE` | Base port for Node.js inspector |
| `CLINE_COMMAND_PERMISSIONS` | JSON policy restricting shell commands (see below) |

### CLINE_COMMAND_PERMISSIONS

Restrict which shell commands the agent can execute:

```bash
export CLINE_COMMAND_PERMISSIONS='{"allow": ["npm *", "git *"], "deny": ["rm -rf *", "sudo *"]}'
```

| Field | Type | Description |
|-------|------|-------------|
| `allow` | `string[]` | Glob patterns for allowed commands. If set, only matching commands are permitted. |
| `deny` | `string[]` | Glob patterns for denied commands. Deny rules always take precedence. |
| `allowRedirects` | `boolean` | Whether to allow shell redirects (`>`, `>>`, `<`). Default: `false`. |

## JSON Output Format

When using `--json`, each message is a JSON object on its own line:

```json
{"type": "say", "text": "I'll create the file now.", "ts": 1760501486669, "say": "text"}
```

| Field | Type | Description |
|-------|------|-------------|
| `type` | `"ask"` or `"say"` | Message category |
| `text` | `string` | Message content |
| `ts` | `number` | Unix timestamp in milliseconds |
| `say` | `string` | Subtype when `type` is `"say"` |
| `ask` | `string` | Subtype when `type` is `"ask"` |
| `reasoning` | `string` | Model reasoning (if available) |
| `partial` | `boolean` | `true` while streaming |

## Configuration Files

```
~/.cline/
  data/
    settings/
      providers.json             # API keys and provider config
      rules/                     # Global rules
      skills/                    # Global skills
    teams/                       # Team state
    sessions/                    # Session database (SQLite)
    logs/
      hub-daemon.log             # Hub logs
  plugins/                       # Global plugins

.cline/                          # Project root
  rules/                         # Project rules
  skills/                        # Project skills
  workflows/                     # Workflows (slash commands)
  hooks/                         # Lifecycle hooks
  plugins/                       # Project plugins
  mcp.json                       # MCP server config
  agents.yaml                    # Agent definitions
```
