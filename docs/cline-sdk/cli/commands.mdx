The Cline CLI provides interactive and headless access to the SDK from the terminal.

## Basic Usage

```bash
# Interactive TUI
cline

# Single prompt (runs agent, exits on completion)
cline "Write a function that sorts an array"

# With a specific model
cline --model anthropic/claude-sonnet-4-6 "Explain this codebase"
```

## Authentication

```bash
# Interactive provider selection and auth
cline auth

# Authenticate a specific provider with API key
cline auth -p anthropic -k "sk-ant-..."

# Set default model during auth
cline auth -p anthropic -k "sk-ant-..." -m anthropic/claude-sonnet-4-6

# Auth for a custom config directory
cline auth -p anthropic -k "sk-ant-..." --config /custom/path
```

## Configuration

```bash
# Open config editor
cline config

# List discovered resources
cline list rules          # Show active rules
cline list skills         # Show available skills
cline list workflows      # Show available workflows
cline list hooks          # Show registered hooks
cline list plugins        # Show loaded plugins
```

## RPC Server

```bash
# Manage the RPC sidecar
cline rpc start           # Start the sidecar
cline rpc stop            # Stop it
cline rpc status          # Check if running
```

## MCP Servers

```bash
# Interactive MCP server management
cline mcp
```

Provides a menu to add, edit, enable/disable, and delete MCP servers. Supports Local (stdio), Remote (SSE), and Remote (HTTP) transports.

## Scheduling

```bash
# Interactive schedule management
cline schedule

# Or create directly with flags
cline schedule create "Task name" \
  --cron "0 9 * * MON-FRI" \
  --prompt "What to do" \
  --workspace /path/to/repo \
  --model anthropic/claude-sonnet-4-6

# Manage schedules
cline schedule list
cline schedule trigger <schedule-id>
cline schedule pause <schedule-id>
cline schedule resume <schedule-id>
cline schedule delete <schedule-id>

# View execution history
cline schedule executions <schedule-id>
```

## Connectors

```bash
# Interactive connector setup wizard
cline connect

# Or connect a specific platform directly with flags
cline connect telegram -m <BOT-ID> -k <BOT-TOKEN>
cline connect slack --token <TOKEN> --signing-secret <SECRET> --base-url <URL>
cline connect discord --app-id <ID> --token <TOKEN> --public-key <KEY> --base-url <URL>
cline connect gchat --credentials <JSON> --base-url <URL>
cline connect whatsapp --phone-id <ID> --token <TOKEN> --app-secret <SECRET> --base-url <URL>
cline connect linear --api-key <KEY> --signing-secret <SECRET> --base-url <URL>

# With access control
cline connect telegram -m <BOT-ID> -k <BOT-TOKEN> \
  --hook-command '/path/to/auth-script.sh'
```

## Session Management

```bash
# List recent sessions
cline history list

# Export session data
cline history export <session-id>

# Manage checkpoints
cline checkpoint list
cline checkpoint delete <checkpoint-id>
```

## Teams

```bash
# Start a team workflow
cline --team-name my-team "Plan and implement feature X"

# Resume team work
cline --team-name my-team "Continue with incomplete tasks"
```

## Diagnostics

```bash
# Run diagnostics
cline doctor
```

Reports on: Node.js version, installed packages, provider configuration, RPC server status, and common issues.

## Global Flags

| Flag | Description |
|------|-------------|
| `--model <id>` | Override the default model (format: `provider/model`) |
| `--team-name <name>` | Enable team mode with the given team name |
| `--session <id>` | Resume an existing session |
| `--config <path>` | Custom config directory (default: `~/.cline`) |
| `--workspace <path>` | Working directory for the agent |
