The Cline SDK is a modular TypeScript framework for building AI agents that can edit files, run commands, browse the web, and use any tool you give them. It powers [Cline CLI](/cline-cli/overview), the VS Code extension, and the desktop app -- and you can embed it in your own applications.

```typescript
import { Agent } from "@clinebot/agents"

const agent = new Agent({
  providerId: "anthropic",
  modelId: "claude-sonnet-4-6",
  systemPrompt: "You are a helpful coding assistant.",
  tools: [],
  onEvent: (event) => {
    if (event.type === "content_update" && event.contentType === "text") {
      process.stdout.write(event.text)
    }
  },
})

const result = await agent.run("Create a hello world Express server")
console.log(result.finishReason) // "completed"
```

## How It Works

The SDK is organized as a layered stack of packages. Use as much or as little as you need:

```
Your Application
       |
  @clinebot/core      Full orchestration: sessions, persistence, config, built-in tools
       |
  @clinebot/agents    Stateless agent loop: run/continue/abort, tool execution, hooks
       |
  @clinebot/llms      LLM gateway: provider abstraction, model catalogs, streaming
       |
  @clinebot/shared    Contracts: types, schemas, tool interfaces, hook engine
```

Most applications start with either `@clinebot/agents` (for lightweight, stateless agent usage) or `@clinebot/core` (for full session management and persistence).

## Installation

<Tabs>
  <Tab title="npm">
    ```bash
    npm install @clinebot/agents @clinebot/llms
    ```
  </Tab>
  <Tab title="bun">
    ```bash
    bun add @clinebot/agents @clinebot/llms
    ```
  </Tab>
  <Tab title="pnpm">
    ```bash
    pnpm add @clinebot/agents @clinebot/llms
    ```
  </Tab>

For full session management, persistence, and built-in tools, also install `@clinebot/core`:

```bash
npm install @clinebot/core
```

Requires Node.js 22 or later and Bun 1.3.10 or later.

## Two Ways to Use the SDK

<Tabs>
  <Tab title="Agent (Lightweight)">
    Use `@clinebot/agents` directly for stateless, in-process agent execution. No persistence, no RPC, no session management. You control the lifecycle.

    ```typescript
    import { Agent } from "@clinebot/agents"

    const agent = new Agent({
      providerId: "anthropic",
      modelId: "claude-sonnet-4-6",
      systemPrompt: "You are a helpful assistant.",
      tools: [myCustomTool],
    })

    const result = await agent.run("Analyze this codebase")
    console.log(result.text)

    // Continue the conversation
    const followUp = await agent.continue("Now refactor the auth module")
    ```
  </Tab>
  <Tab title="ClineCore (Full Stack)">
    Use `@clinebot/core` for session persistence, built-in tools (bash, file editing, web fetch), config watching, and optional RPC for multi-process architectures.

    ```typescript
    import { ClineCore } from "@clinebot/core"

    const cline = await ClineCore.create({
      clientName: "my-app",
    })

    const session = await cline.start({
      prompt: "Set up a CI pipeline for this project",
      config: {
        providerId: "anthropic",
        modelId: "claude-sonnet-4-6",
        apiKey: process.env.ANTHROPIC_API_KEY,
        cwd: "/path/to/project",
      },
    })

    console.log(session.result?.text)
    ```
  </Tab>
</Tabs>

## Supported Providers

The SDK supports all major LLM providers through `@clinebot/llms`:

| Provider | Models | Streaming | Vision | Tool Use |
|----------|--------|-----------|--------|----------|
| Anthropic | Claude 4, 3.5 | Yes | Yes | Yes |
| OpenAI | GPT-4o, o1 | Yes | Yes | Yes |
| Google | Gemini 2.5 | Yes | Yes | Yes |
| AWS Bedrock | Claude, Llama | Yes | Yes | Yes |
| Mistral | Mistral Large | Yes | No | Yes |
| OpenAI-compatible | Any | Yes | Varies | Varies |

Configure providers via environment variables, settings files, or runtime config. See [Providers and Models](/cline-sdk/providers-and-models) for details.
