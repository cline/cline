仓库一大，最糟糕的读法就是“从头到尾全看一遍”。最快的读法，是先知道自己该站在哪里。这一页就是给你定站位的。

## 如果你只读 12 个文件，就读这 12 个

| 文件 | 为什么重要 |
|---|---|
| `src/extension.ts` | VS Code 启动顺序、命令注册、存储迁移、Webview 注册都在这里 |
| `src/hosts/vscode/VscodeWebviewProvider.ts` | React 前端与扩展宿主的消息桥 |
| `src/core/controller/index.ts` | 共享服务所有权、任务创建、远程配置、MCP 初始化 |
| `src/core/task/index.ts` | 主任务循环和任务级资源绑定 |
| `src/core/task/ToolExecutor.ts` | 审批、校验、安全限制、工具结果回灌 |
| `src/core/task/tools/ToolExecutorCoordinator.ts` | 从工具名到 handler 的类型化路由 |
| `src/core/task/loop-detection.ts` | 重复工具调用安全逻辑 |
| `src/core/task/TaskPresentationScheduler.ts` | 流式 UI 调度器 |
| `src/services/mcp/McpHub.ts` | MCP 设置监听、传输建立、重连逻辑 |
| `src/core/storage/StateManager.ts` | 缓存状态、debounce 持久化、同步策略 |
| `webview-ui/src/services/grpc-client-base.ts` | Webview 请求与流式响应传输 |
| `cli/src/agent/ClineAgent.ts` | ACP 集成、多会话 CLI 嵌入 |

## 从症状反查源码

| 现象 | 先看哪里 | 原因 |
|---|---|---|
| 侧边栏不更新 | `src/hosts/vscode/VscodeWebviewProvider.ts`, `webview-ui/src/services/grpc-client-base.ts` | 请求和流式响应边界都在这里 |
| 工具一直重复调用 | `src/core/task/ToolExecutor.ts`, `src/core/task/loop-detection.ts` | 审批与循环检测都集中在这里 |
| MCP 服务连不上 | `src/services/mcp/McpHub.ts`, `src/services/mcp/schemas.ts` | 校验、文件监听、传输建立都在这里 |
| CLI 和 VS Code 行为不同 | `cli/src/index.ts`, `cli/src/agent/ClineAgent.ts`, `src/hosts/external/ExternalWebviewProvider.ts` | 宿主差异应该在这层，而不是在 Task 引擎里 |
| 历史或检查点不对劲 | `src/core/storage/StateManager.ts`, `src/core/storage/disk.ts`, `src/integrations/checkpoints/factory.ts` | 存储与检查点策略在这几处交界 |

```text

2. src/core/controller/index.ts
3. src/core/task/index.ts
4. src/core/task/ToolExecutor.ts
5. src/core/task/tools/ToolExecutorCoordinator.ts
6. src/core/task/loop-detection.ts
7. src/core/task/TaskPresentationScheduler.ts
8. src/services/mcp/McpHub.ts
9. src/core/storage/StateManager.ts
```
