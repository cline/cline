When a repository is large, the fastest path is not “read everything.” The fastest path is “know where to stand.” This map tells you where to stand.

## If you only read 12 files, read these

| File | Why it matters |
|---|---|
| `src/extension.ts` | VS Code boot sequence, command registration, storage migration, and webview registration |
| `src/hosts/vscode/VscodeWebviewProvider.ts` | message bridge between React and the extension host |
| `src/core/controller/index.ts` | service ownership, task creation, remote config, and MCP setup |
| `src/core/task/index.ts` | main conversation loop and task-owned services |
| `src/core/task/ToolExecutor.ts` | approval, validation, safety, and tool result plumbing |
| `src/core/task/tools/ToolExecutorCoordinator.ts` | typed routing from tool names to handlers |
| `src/core/task/loop-detection.ts` | repeated-tool safety logic |
| `src/core/task/TaskPresentationScheduler.ts` | streaming UI scheduling |
| `src/services/mcp/McpHub.ts` | MCP settings watching, transport setup, reconnect logic |
| `src/core/storage/StateManager.ts` | cached state, debounce, and persistence rules |
| `webview-ui/src/services/grpc-client-base.ts` | webview request and streaming response transport |
| `cli/src/agent/ClineAgent.ts` | ACP integration and multi-session CLI embedding |

## Debug from symptom to source

| Symptom | Start here | Why |
|---|---|---|
| Sidebar UI is not receiving updates | `src/hosts/vscode/VscodeWebviewProvider.ts`, `webview-ui/src/services/grpc-client-base.ts` | this is the request and streaming response boundary |
| A tool keeps repeating | `src/core/task/ToolExecutor.ts`, `src/core/task/loop-detection.ts` | approval and loop detection live here |
| MCP server will not connect | `src/services/mcp/McpHub.ts`, `src/services/mcp/schemas.ts` | validation, file watching, and transport setup are centralized there |
| CLI behaves differently from VS Code | `cli/src/index.ts`, `cli/src/agent/ClineAgent.ts`, `src/hosts/external/ExternalWebviewProvider.ts` | host differences should appear here, not in the task engine |
| Task history or checkpoint data looks wrong | `src/core/storage/StateManager.ts`, `src/core/storage/disk.ts`, `src/integrations/checkpoints/factory.ts` | storage and checkpoint policy are separated but adjacent |
