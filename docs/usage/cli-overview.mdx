## Prerequisites

- Cline CLI installed ([Install Guide](/getting-started/installing-cline))
- Provider authenticated (`clite auth`) ([Authorization Guide](/getting-started/authorizing-with-cline#cli-setup))

## Quick Start

```bash
# Interactive session
clite

# Run one task immediately
clite "refactor this module to use async/await"

# Structured output for scripts
clite --json "list TODO comments"
```

## Headless mode

Use headless mode for scripts/automation and processable output.

Headless is triggered when using flags like `--json`, when stdin is piped, or when output is redirected.

#### When headless activates

| Invocation | Reason |
|---|---|
| `clite --json "task"` | JSON output mode |
| `cat file \| clite "task"` | stdin is piped |
| `clite "task" > output.txt` | stdout is redirected |

```bash
# CI/script style execution
git diff | clite "review these changes"

# JSON for parsing
clite --json "summarize this changelog" | jq -r '.text'
```

See: [Headless Mode](#headless-mode)

### Autonomous execution

For fully unattended runs, use auto-approval:

```bash
clite --auto-approve true "run tests and fix failures"
```

Mode selection also works in headless runs:

```bash
# plan-first
clite -p "design migration plan"

# act immediately (default)
clite "apply migration"
```

<Warning>
Autonomous execution can modify files and run commands without further prompts. Use a clean branch and review results.
</Warning>

## High-Value Commands

```bash
clite --help
clite <command> --help
```

| Command | Purpose |
|---|---|
| `clite` | Start interactive mode or run a prompt |
| `clite auth` | Authenticate and set provider/model |
| `clite config` | Open the interactive config view |
| `clite mcp` | Manage MCP servers |
| `clite doctor` | Diagnose/fix configuration issues |
| `clite history` | Show and manage task history |
| `clite schedule` | Manage scheduled tasks |
| `clite hub` | Manage local hub daemon |
| `clite kanban` | Launch Kanban app |

## Most-Used Global Flags

Source of truth: [CLI Reference](/cli/cli-reference)

| Flag | Purpose |
|---|---|
| `-p, --plan` | Start in Plan mode |
| `--auto-approve <boolean>` | Global tool auto-approval (`true`/`false`, default `true`) |
| `-m, --model <model>` | Override model for this run |
| `-P, --provider <id>` | Override provider for this run |
| `-c, --cwd <path>` | Set working directory |
| `--config <dir>` | Use config directory |
| `--data-dir <dir>` | Use isolated local state |
| `--json` | Output newline-delimited JSON messages |
| `--thinking <level>` | Set reasoning effort: `none\|low\|medium\|high\|xhigh` (default `medium`) |
| `-t, --timeout <seconds>` | Set task timeout |

## Automation Patterns

### Pipe context in

```bash
cat README.md | clite "summarize key setup steps"
git diff | clite "review for potential regressions"
```

### Chain tasks

```bash
git diff | clite "explain these changes" | clite "write a commit message"
```

### Restrict command execution

```bash
export CLINE_COMMAND_PERMISSIONS='{"allow": ["npm *", "git *"], "deny": ["rm -rf *", "sudo *"]}'
```

### Include images in tasks

```bash
clite -i "fix the layout issue shown in @./screenshot.png"

# or reference inline
clite "fix the UI shown in @./design-mockup.png"
```

### Set execution timeout

```bash
clite --timeout 600 "run full test suite"
```

## JSON Output Schema

When using `--json`, each line is a JSON message object.

```json
{"type":"say","text":"I'll create the file now.","ts":1760501486669,"say":"text"}
```

| Field | Type | Description |
|---|---|---|
| `type` | `"ask"` or `"say"` | Message category |
| `text` | `string` | Message content |
| `ts` | `number` | Unix timestamp in milliseconds |
| `say` | `string` | Subtype when `type` is `"say"` |
| `ask` | `string` | Subtype when `type` is `"ask"` |
| `reasoning` | `string` | Optional model reasoning |
| `partial` | `boolean` | Streaming flag |
