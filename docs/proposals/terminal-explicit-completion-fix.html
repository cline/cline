Cline Terminal Explicit Completion Fix

Proposal and test plan for stopping one-shot terminal runs from ending when a model emits a text-only "done" message instead of a terminal completion tool call.

worktree: /tmp/cline-terminal-explicit-completion branch: fix/terminal-explicit-completion scope: terminal CLI, one-shot Act runs

Problem

When using the Cline provider with GPT models, the assistant sometimes responds with a normal text comment such as "I am done" or "The task is complete" and does not produce a tool call. In the terminal one-shot path, that no-tool assistant turn can be treated as a completed run, so the user has to nudge the model to continue or properly finish.

Claude-family models tend to call the expected terminal completion tool more reliably, so the behavior is less visible there. The root issue is not that the Cline provider lacks tool support. The issue is that the terminal Act path did not expose an explicit completion tool, so the runtime had no semantic completion signal to require.

Root Cause

Layer Current behavior Impact
Model/provider The model may emit text instead of a tool call. There is no provider-level end_turn signal in this path.
Runtime Already supports completionPolicy.requireCompletionTool when a tool has lifecycle.completesRun. The guard works only if a completion tool is actually present.
CLI Act mode Passed a submit executor but used the Act preset, which did not enable submit_and_exit. No-tool assistant text could finish the run.

Fix

Change: one-shot terminal runs opt into enableSubmitAndExit. The runtime builder then exposes submit_and_exit, and the existing completion policy requires that tool before considering the run complete.

This keeps the fix in the terminal runtime contract instead of adding model-specific prompt nudges. If the model emits text without a tool call, the runtime adds a system reminder and continues the run. If the task is actually complete, the model must call submit_and_exit.

Files changed

  • apps/cli/src/runtime/run-agent.ts: sets enableSubmitAndExit: true for one-shot terminal runs.
  • sdk/packages/core/src/types/config.ts: adds enableSubmitAndExit to core runtime features.
  • sdk/packages/core/src/runtime/orchestration/runtime-builder.ts: threads the flag into built-in tool construction.
  • Unit tests cover the CLI start config and Act-mode runtime tool selection.
Known tradeoff: the existing built-in tools factory excludes ask_question when submit_and_exit is enabled. This patch preserves that existing behavior to keep the change narrow.

Expected Result

  • The model should see submit_and_exit as an available tool.
  • If GPT replies with text-only completion, the run should continue after a reminder instead of ending.
  • The run should end only after the model calls submit_and_exit.
