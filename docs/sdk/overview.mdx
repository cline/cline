The Cline SDK is a modular TypeScript framework for building AI agents that can edit files, run commands, browse the web, and use any tool you give them. It powers [Cline CLI](/usage/cli-overview), the VS Code extension, and the JetBrains plugin -- and you can embed it in your own applications.

```typescript
import { Agent } from "@clinebot/sdk"

const agent = new Agent({
  providerId: "anthropic",
  modelId: "claude-sonnet-4-6",
  systemPrompt: "You are a helpful coding assistant.",
  tools: [],
  onEvent: (event) => {
    if (event.type === "content_update" && event.contentType === "text") {
      process.stdout.write(event.text)
    }
  },
})

const result = await agent.run("Create a hello world Express server")
console.log(result.finishReason) // "completed"
```

`@clinebot/sdk` includes everything you need -- agents, tools, sessions, providers, and the full built-in tool suite:

```typescript
import { Agent, ClineCore, createTool, type AgentPlugin } from "@clinebot/sdk"
```

Under the hood, the SDK is composed of four packages (`@clinebot/core`, `@clinebot/agents`, `@clinebot/llms`, `@clinebot/shared`). You can install these individually if you want a minimal dependency footprint -- for example, `@clinebot/agents` gives you just the stateless agent loop without sessions or built-in tools. See [Architecture Overview](/sdk/architecture/overview) for package details.

Requires Node.js 22 or later and Bun 1.3.10 or later.

## How It Works

The SDK is organized as a layered stack of packages. Use as much or as little as you need:

```
Your Application
       |
  @clinebot/core      Full orchestration: sessions, persistence, config, built-in tools
       |
  @clinebot/agents    Stateless agent loop: run/continue/abort, tool execution, hooks
       |
  @clinebot/llms      LLM gateway: provider abstraction, model catalogs, streaming
       |
  @clinebot/shared    Contracts: types, schemas, tool interfaces, hook engine
```

Most applications just install `@clinebot/sdk` and import everything from there. If you only need the lightweight agent loop, you can install `@clinebot/agents` directly.

## Two Ways to Use the SDK

<Tabs>
  <Tab title="Agent (Lightweight)">
    Use `@clinebot/agents` directly for stateless, in-process agent execution. No persistence, no hub, no session management. You control the lifecycle.

    ```typescript
    import { Agent } from "@clinebot/agents"

    const agent = new Agent({
      providerId: "anthropic",
      modelId: "claude-sonnet-4-6",
      systemPrompt: "You are a helpful assistant.",
      tools: [myCustomTool],
    })

    const result = await agent.run("Analyze this codebase")
    console.log(result.text)

    // Continue the conversation
    const followUp = await agent.continue("Now refactor the auth module")
    ```
  </Tab>
  <Tab title="ClineCore (Full Stack)">
    Use `ClineCore` for session persistence, built-in tools (bash, file editing, web fetch), config watching, and optional hub for multi-process architectures.

    ```typescript
    import { ClineCore } from "@clinebot/sdk"

    const cline = await ClineCore.create({
      clientName: "my-app",
    })

    const session = await cline.start({
      prompt: "Set up a CI pipeline for this project",
      config: {
        providerId: "anthropic",
        modelId: "claude-sonnet-4-6",
        apiKey: process.env.ANTHROPIC_API_KEY,
        cwd: "/path/to/project",
      },
    })

    console.log(session.result?.text)
    ```
  </Tab>
</Tabs>

## Supported Providers

The SDK supports all providers available under [AI SDK](https://ai-sdk.dev/docs/foundations/providers-and-models).

Configure providers via environment variables, settings files, or runtime config. See [Model Providers](/sdk/model-providers) for details.
