An `Agent` is the core execution unit of the Cline SDK. It runs an iterative loop: send messages to an LLM, execute any tool calls the model makes, feed results back, and repeat until the task is complete.

The `Agent` class from `@clinebot/agents` is stateless -- it has no persistence, no session management, and no built-in tools. You provide everything it needs through configuration. For a batteries-included experience with persistence and built-in tools, see [ClineCore](/sdk/sessions).

## Creating an Agent

```typescript
import { Agent } from "@clinebot/agents"

const agent = new Agent({
  providerId: "anthropic",
  modelId: "claude-sonnet-4-6",
  apiKey: process.env.ANTHROPIC_API_KEY,
  systemPrompt: "You are a helpful coding assistant.",
  tools: [],
})
```

### Configuration Options

| Option | Type | Description |
|--------|------|-------------|
| `providerId` | `string` | LLM provider (`"anthropic"`, `"openai"`, `"google"`, etc.) |
| `modelId` | `string` | Model identifier (e.g., `"claude-sonnet-4-6"`) |
| `apiKey` | `string` | Provider API key |
| `systemPrompt` | `string` | Instructions that guide agent behavior |
| `tools` | `Tool[]` | Tools the agent can call |
| `extensions` | `AgentExtension[]` | Plugins that extend agent capabilities |
| `maxIterations` | `number` | Maximum loop iterations before stopping (default: 50) |
| `maxParallelToolCalls` | `number` | Max concurrent tool executions (default: 1) |
| `apiTimeoutMs` | `number` | Timeout for each LLM API call |
| `maxTokensPerTurn` | `number` | Token limit per model response |
| `initialMessages` | `Message[]` | Pre-seed the conversation history |
| `toolPolicies` | `Record<string, ToolPolicy>` | Per-tool approval and enablement rules |
| `onEvent` | `(event: AgentEvent) => void` | Callback for real-time events |

## Lifecycle

Every agent follows this execution flow:

```
run() or continue()
  -> session_start hook
    -> run_start hook
      -> [LOOP: iteration 1..maxIterations]
        -> iteration_start hook
        -> LLM API call (with tools)
        -> Tool execution (parallel if configured)
        -> Tool results fed back to model
        -> iteration_end hook
        -> [Check: no tool calls = task complete]
      -> [Loop exits]
    -> run_end hook
  -> session_shutdown hook
```

The loop exits when one of these conditions is met:

| Finish Reason | Meaning |
|---------------|---------|
| `completed` | Model responded without making any tool calls (task is done) |
| `max_iterations` | Hit the `maxIterations` limit |
| `mistake_limit` | Too many consecutive tool failures |
| `aborted` | `agent.abort()` was called |
| `error` | An unrecoverable error occurred |

## Running an Agent

### First Run

`run()` starts a new conversation with a user message:

```typescript
const result = await agent.run("Create a fibonacci function in TypeScript")
```

### Continuing a Conversation

`continue()` adds a new user message to the existing conversation:

```typescript
const r1 = await agent.run("What does this codebase do?")
const r2 = await agent.continue("Can you refactor the auth module?")
const r3 = await agent.continue("Add tests for the changes you made")
```

The agent retains the full conversation history between calls, so each `continue()` has the full context of everything that came before.

### The AgentResult

Both `run()` and `continue()` return an `AgentResult`:

```typescript
interface AgentResult {
  text: string               // The agent's final text response
  finishReason: string       // Why the agent stopped
  iterations: number         // How many loop iterations it took
  usage: AgentUsage          // Token counts and cost
  messages: Message[]        // Full conversation history
  toolCalls: ToolCallRecord[] // All tool calls made during this run
}
```

```typescript
const result = await agent.run("Explain async/await")

console.log(result.text)           // The explanation
console.log(result.finishReason)   // "completed"
console.log(result.iterations)     // 1 (no tools, single response)
console.log(result.usage)          // { inputTokens: 150, outputTokens: 500, ... }
```

## Aborting and Shutting Down

### Abort

Cancel the current run. The agent stops after the current tool execution completes:

```typescript
// From another context (timeout, user cancellation, etc.)
agent.abort("User cancelled the operation")
```

The `run()` or `continue()` call resolves with `finishReason: "aborted"`.

### Shutdown

Clean up all resources. Call this when you're done with the agent:

```typescript
await agent.shutdown("Application closing")
```

After shutdown, the agent cannot be used again.

## Event Handling

Subscribe to real-time events during execution:

```typescript
const agent = new Agent({
  // ...config
  onEvent: (event) => {
    switch (event.type) {
      case "content_start":
        if (event.contentType === "text") console.log("[text start]")
        if (event.contentType === "tool") console.log(`[tool: ${event.toolName}]`)
        break
      case "content_update":
        if (event.contentType === "text") process.stdout.write(event.text)
        break
      case "content_end":
        console.log("[content end]")
        break
      case "iteration_start":
        console.log(`[iteration ${event.iteration}]`)
        break
      case "usage":
        console.log(`[tokens: ${event.inputTokens} in, ${event.outputTokens} out]`)
        break
      case "done":
        console.log(`[done: ${event.finishReason}]`)
        break
      case "error":
        console.error(`[error: ${event.message}]`)
        break
    }
  },
})
```

You can also subscribe after creation:

```typescript
const unsubscribe = agent.subscribeEvents((event) => {
  // handle events
})

// Later, stop listening
unsubscribe()
```

See [Streaming and Events](/sdk/streaming-and-events) for the complete event reference.

## Managing Conversation History

### Read Messages

```typescript
const messages = agent.getMessages()
// Returns the full conversation as Message[] (system, user, assistant, tool results)
```

### Clear History

```typescript
agent.clearHistory()
// Resets the conversation. Next call must be run(), not continue().
```

### Restore History

Pre-load a conversation from a previous session:

```typescript
const savedMessages = loadFromDatabase() // your persistence layer
agent.restore(savedMessages)

// Now continue() picks up where the previous session left off
const result = await agent.continue("What were we working on?")
```

## Error Recovery

The agent tracks consecutive mistakes (tool failures, API errors, invalid tool calls). When a tool succeeds, the counter resets. When it hits the mistake limit, the agent stops with `finishReason: "mistake_limit"`.

The agent also detects loops -- if the same tool call is made repeatedly with identical arguments, it will warn the model and eventually force-stop execution.

## Parallel Tool Execution

By default, tools run one at a time. Set `maxParallelToolCalls` to run multiple tools concurrently:

```typescript
const agent = new Agent({
  // ...config
  maxParallelToolCalls: 3,
})
```

This is useful when the model makes multiple independent tool calls in a single response (e.g., reading several files at once).

## Updating Configuration

Change the model or provider mid-session:

```typescript
agent.updateConnection({
  providerId: "openai",
  modelId: "gpt-4o",
  apiKey: process.env.OPENAI_API_KEY,
})
```

The conversation history is preserved. Only the LLM connection changes.
