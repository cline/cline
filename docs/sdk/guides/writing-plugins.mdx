Plugins are the primary way to package reusable agent capabilities. This guide walks through building a production-quality plugin from scratch.

## What You'll Build

A GitHub integration plugin that:
- Registers tools for interacting with GitHub (list issues, create PRs, post comments)
- Logs all tool calls for auditing
- Tracks token usage per session

## Step 1: Define the Plugin Structure

```typescript
// github-plugin.ts
import { type AgentExtension, type AgentResult } from "@clinebot/core"
import { createTool } from "@clinebot/agents"

interface GitHubConfig {
  token: string
  owner: string
  repo: string
}

export function createGitHubPlugin(config: GitHubConfig): AgentExtension {
  let totalTokens = 0

  return {
    name: "github-integration",
    manifest: {
      capabilities: ["tools", "hooks"],
      hookStages: ["run_start", "tool_call_before", "run_end"],
    },

    setup(api) {
      // Register tools in the setup phase
      api.registerTool(createListIssuesTool(config))
      api.registerTool(createCreateIssueTool(config))
      api.registerTool(createPostCommentTool(config))
    },

    onRunStart({ userMessage }) {
      console.log(`[github] Run started: "${userMessage.slice(0, 60)}..."`)
    },

    onToolCall({ call }) {
      console.log(`[github] Tool: ${call.name}(${JSON.stringify(call.input).slice(0, 100)})`)
    },

    onRunEnd({ result }) {
      totalTokens += result.usage.inputTokens + result.usage.outputTokens
      console.log(`[github] Run complete. Session tokens so far: ${totalTokens}`)
    },
  }
}
```

## Step 2: Create the Tools

```typescript
function createListIssuesTool(config: GitHubConfig) {
  return createTool({
    name: "list_github_issues",
    description: `List open issues in ${config.owner}/${config.repo}. Returns issue numbers, titles, labels, and assignees.`,
    inputSchema: {
      type: "object",
      properties: {
        state: {
          type: "string",
          enum: ["open", "closed", "all"],
          description: "Issue state filter. Default: open.",
        },
        labels: {
          type: "string",
          description: "Comma-separated label names to filter by (e.g., 'bug,priority:high').",
        },
        limit: {
          type: "number",
          description: "Maximum issues to return. Default: 10, max: 100.",
        },
      },
    },
    execute: async (input) => {
      const params = new URLSearchParams({
        state: input.state ?? "open",
        per_page: String(Math.min(input.limit ?? 10, 100)),
      })
      if (input.labels) params.set("labels", input.labels)

      const response = await fetch(
        `https://api.github.com/repos/${config.owner}/${config.repo}/issues?${params}`,
        { headers: { Authorization: `token ${config.token}` } }
      )

      const issues = await response.json()
      return {
        issues: issues.map((i: Record<string, unknown>) => ({
          number: i.number,
          title: i.title,
          state: i.state,
          labels: (i.labels as Array<{ name: string }>).map((l) => l.name),
          assignee: (i.assignee as { login: string } | null)?.login,
          createdAt: i.created_at,
        })),
        total: issues.length,
      }
    },
  })
}

function createCreateIssueTool(config: GitHubConfig) {
  return createTool({
    name: "create_github_issue",
    description: `Create a new issue in ${config.owner}/${config.repo}.`,
    inputSchema: {
      type: "object",
      properties: {
        title: { type: "string", description: "Issue title" },
        body: { type: "string", description: "Issue body (Markdown supported)" },
        labels: {
          type: "array",
          items: { type: "string" },
          description: "Labels to apply",
        },
      },
      required: ["title"],
    },
    execute: async (input) => {
      const response = await fetch(
        `https://api.github.com/repos/${config.owner}/${config.repo}/issues`,
        {
          method: "POST",
          headers: {
            Authorization: `token ${config.token}`,
            "Content-Type": "application/json",
          },
          body: JSON.stringify({
            title: input.title,
            body: input.body,
            labels: input.labels,
          }),
        }
      )

      const issue = await response.json()
      return { number: issue.number, url: issue.html_url }
    },
  })
}

function createPostCommentTool(config: GitHubConfig) {
  return createTool({
    name: "post_github_comment",
    description: `Post a comment on an issue or PR in ${config.owner}/${config.repo}.`,
    inputSchema: {
      type: "object",
      properties: {
        issueNumber: { type: "number", description: "Issue or PR number" },
        body: { type: "string", description: "Comment body (Markdown supported)" },
      },
      required: ["issueNumber", "body"],
    },
    execute: async (input) => {
      const response = await fetch(
        `https://api.github.com/repos/${config.owner}/${config.repo}/issues/${input.issueNumber}/comments`,
        {
          method: "POST",
          headers: {
            Authorization: `token ${config.token}`,
            "Content-Type": "application/json",
          },
          body: JSON.stringify({ body: input.body }),
        }
      )

      const comment = await response.json()
      return { id: comment.id, url: comment.html_url }
    },
  })
}
```

## Step 3: Use the Plugin

```typescript
import { Agent } from "@clinebot/agents"
import { createGitHubPlugin } from "./github-plugin"

const agent = new Agent({
  providerId: "anthropic",
  modelId: "claude-sonnet-4-6",
  apiKey: process.env.ANTHROPIC_API_KEY,
  systemPrompt: "You are a project manager assistant with access to GitHub.",
  tools: [],
  plugins: [
    createGitHubPlugin({
      token: process.env.GITHUB_TOKEN,
      owner: "my-org",
      repo: "my-project",
    }),
  ],
})

await agent.run("List all open bugs and create a summary issue with the count")
```

## Distributing as a File Plugin

To load this plugin from a file in ClineCore, pass its path in `pluginPaths`:

```typescript
// /absolute/path/to/github.ts
import { type AgentExtension, createTool } from "@clinebot/core"

const plugin: AgentExtension = {
  name: "github",
  manifest: { capabilities: ["tools"] },
  setup(api) {
    api.registerTool(
      createTool({
        name: "list_github_issues",
        // ... tool definition
      })
    )
  },
}

export default plugin
```

Then include it in session config:

```typescript
await cline.start({
  prompt: "Use the GitHub plugin",
  config: {
    // ...model/runtime config
    pluginPaths: ["/absolute/path/to/github.ts"],
  },
})
```

## Distributing via CLI Install

To make your plugin installable with `clite plugin install`, add a `cline` field to your `package.json`:

```json
{
  "name": "cline-github-plugin",
  "version": "1.0.0",
  "type": "module",
  "cline": {
    "plugins": [
      {
        "paths": ["./github-plugin.ts"],
        "capabilities": ["tools", "hooks"]
      }
    ]
  },
  "peerDependencies": {
    "@cline/core": "*"
  },
  "peerDependenciesMeta": {
    "@cline/core": {
      "optional": true
    }
  }
}
```

The `cline.plugins` array tells the installer which files are plugin entry points. Each entry can be:

- An object with `paths` (array of `.ts`/`.js` files) and optional `capabilities`
- A plain string path like `"./index.ts"`

Dependencies under the `@cline/` scope are provided by the host runtime and are stripped automatically during install, so declare them as peer dependencies.

Users install your plugin with:

```bash
# From a git repository
clite plugin install https://github.com/your-org/cline-github-plugin.git

# From npm
clite plugin install npm:cline-github-plugin

# From a local checkout
clite plugin install ./cline-github-plugin
```

After installation, the plugin is automatically available in all sessions. Users can verify it appears in `clite config` under the plugin tab.

If your `package.json` omits the `cline` field, the installer falls back to auto-discovering `.ts` and `.js` files in the package root (excluding `node_modules` and `.git`). Declaring explicit paths is recommended for clarity.

## Plugin Design Guidelines

1. Use factory functions (like `createGitHubPlugin`) when the plugin needs configuration. Export the plugin object directly when it doesn't.

2. Keep `setup()` synchronous and fast. It runs before the first LLM call, so any async initialization delays the agent.

3. Register all tools in `setup()`, not in lifecycle hooks. Tools must be available before the first iteration.

4. Use lifecycle hooks for observation (logging, metrics, auditing), not for modifying agent behavior. If you need to modify behavior, consider using the `before_agent_start` hook to adjust the system prompt.

5. Handle errors gracefully in hooks. A thrown error in `onToolCall` will count as a tool failure. If your hook is purely observational, catch errors internally.
