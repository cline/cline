두 가지 개념을 먼저 이해하세요:

**작업(Task)** - Cline이 완료해야 할 단일 작업("utils.js에 테스트 추가" 등)입니다. 원하는 내용을 설명하면 Cline이 계획을 세우고 실행합니다. 작업은 인스턴스에서 실행됩니다.

**인스턴스(Instance)** - 독립적인 Cline 작업 공간입니다. 각 인스턴스는 한 번에 하나의 작업만 실행합니다. 프로젝트의 다른 부분을 병렬로 작업하려면 여러 인스턴스를 생성하세요.

## 1. 대화형 모드: 먼저 계획, 그다음 실행

Cline이 어떻게 동작하는지 확인하려면 이 모드부터 시작하세요. 대화형 모드는 실행 전에 계획을 검토할 수 있는 채팅 세션을 엽니다.

```bash
cline
```

Cline이 현재 디렉터리에서 대화형 세션을 시작합니다. 작업을 메시지로 입력하면 Cline이 Plan 모드로 들어가 단계별 전략을 제안합니다.

채팅에서 계획을 검토하거나 수정한 뒤, 실행할 준비가 되면 다음을 입력하세요:

```bash
/act
```

Cline이 승인된 단계들을 실행합니다(파일 읽기, 코드 작성, 명령 실행). 과정 전체를 사용자가 통제합니다.

## 2. 헤드리스 단발성 실행: 채팅 없이 작업 완료

자동화가 필요할 때, 한 줄로 작업을 끝내고 싶다면 이 모드를 사용하세요.

```bash
cline instance new --default
cline task new -y "모든 Go 파일에 대한 단위 테스트 생성"
```

`-y`(YOLO) 플래그를 사용하면 Cline이 대화 없이 자율적으로 계획하고 실행합니다. CI, cron 작업, 스크립트에 적합합니다.

예시:

```bash
# 완전한 기능 생성
cline task new -y "사용자 인증을 위한 REST API 만들기"

# 문서 생성
cline task new -y "src/의 모든 함수에 JSDoc 주석 추가"

# 코드 리팩터링
cline task new -y "var 선언을 모두 const/let으로 변환"
```

작업 모니터링:

```bash
# 작업 상태 보기
cline task view

# 작업 진행 상황을 실시간으로 추적
cline task view --follow
```

Ctrl+C로 보기에서 나올 수 있습니다.

<Note>
YOLO 모드는 감독 대신 속도를 얻는 방식입니다. 안전하게 되돌릴 수 있도록 디렉터리나 깨끗한 Git 브랜치에서 실행하세요.
</Note>

## 3. 멀티 인스턴스: 에이전트 병렬 실행

여러 인스턴스를 사용하면 동일한 프로젝트에서 컨텍스트 충돌 없이 병렬 작업이 가능합니다. 프런트엔드, 백엔드, 인프라 작업을 동시에 진행할 수 있습니다.

첫 인스턴스를 생성하세요:

```bash
cline instance new
```

인스턴스 주소가 반환되며, 이 주소로 작업을 대상 지정합니다. 이 인스턴스에 작업을 붙입니다:

```bash
# 첫 인스턴스에서 프런트엔드 작업
cline task new -y "React 컴포넌트 구축"
```

두 번째 인스턴스를 생성하고 기본으로 설정:

```bash
cline instance new --default
```

이제 주소를 지정하지 않아도 기본 인스턴스를 사용합니다:

```bash
# 새로운 기본 인스턴스에서 백엔드 작업
cline task new -y "API 엔드포인트 구현"
```

실행 중인 인스턴스 목록:

```bash
cline instances list
```

작업이 끝나면 모든 인스턴스 종료:

```bash
cline instances kill -a
```

<Tip>
`cline instance new`가 반환한 인스턴스 주소를 잘 기록해 두세요. 여러 에이전트를 스크립트로 운용할 때 이 ID를 저장해 작업을 해당 인스턴스로 보내면 됩니다.
</Tip>

## 로컬 제공자 컨텍스트 윈도우 설정

Ollama와 LM Studio는 CLI로 모델 컨텍스트 윈도우를 설정할 수 있습니다:

```bash
# Ollama용
cline config s ollama-api-options-ctx-num=32768

# LM Studio용
cline config s lm-studio-max-tokens=32768
```

다른 제공자(Anthropic, OpenRouter 등)는 모델 메타데이터에 컨텍스트 크기가 정의되어 있어 사용자가 설정할 수 없습니다. Cline은 각 모델의 기본 컨텍스트 제한을 자동으로 사용합니다.

## 올바른 흐름 선택하기

- **대화형 모드**: 새로운 문제를 탐색하거나 Cline 동작을 학습할 때, 또는 실행 전에 계획을 검토하고 싶을 때
- **헤드리스 단발성 실행**: 자동화, CI/CD, 감독 없이 Cline 실행을 신뢰할 수 있는 작업에 적합
- **멀티 인스턴스**: 작업을 병렬화하거나 프로젝트의 다른 영역에 대해 컨텍스트를 분리해야 할 때

<Tip>
명령과 플래그의 상세 정보는 [CLI 레퍼런스](/cline-cli/cli-reference)에서 확인하세요.
</Tip>
