# Self Improvement

## Purpose

`meta-knowledge` can improve itself, but it should do so with discipline.

## When Self-Improvement Is Justified

Self-improvement is justified when:

- the skill repeatedly misses the right target
- the skill creates too much overhead
- the skill does not preserve enough evidence
- the skill cannot easily support active project workflows

## Guardrails

- Do not redesign the whole skill from one weak signal.
- Prefer additive improvements before structural rewrites.
- Record the evidence in the knowledge base before patching the skill.
- Keep the trigger conditions narrow and explicit.

## Preferred Self-Improvement Order

1. templates
2. references
3. scripts
4. main `SKILL.md`

This order keeps the main skill stable unless the workflow itself is wrong.
