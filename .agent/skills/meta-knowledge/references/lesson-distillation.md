# Lesson Distillation

## Purpose

This reference explains how to turn raw notes into reusable lessons.

## Distillation Questions

For each raw observation, ask:

1. What exactly went wrong or worked unusually well?
2. Is this a one-off case or likely to recur?
3. Does the lesson belong to execution, documentation, validation, or routing?
4. Would changing a skill or rule prevent future repetition?
5. Is the lesson local to one skill or broadly reusable?

## Lesson Categories

### Execution Lesson

Examples:

- wrong task order
- missing stage gate
- missing helper-skill usage

### Documentation Lesson

Examples:

- artifacts drift from real status
- missing template sections
- missing handoff notes

### Validation Lesson

Examples:

- tests happen too late
- validation output is hard to reuse
- blocked validation is not clearly recorded

### Routing Lesson

Examples:

- discovery findings should have updated INCEPTION earlier
- skill improvements were patched locally but not generalized

## Keep Or Discard

Keep the lesson if at least one is true:

- it recurred
- the user explicitly corrected it
- it affects multiple future tasks
- it changes completion quality in a meaningful way

Discard or defer it if:

- it is too case-specific
- it has no clear target
- the change cost exceeds the likely future benefit
